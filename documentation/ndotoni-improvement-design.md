# Ndotoni Acquisition Landing Pages Design Document

## Overview

Ndotoni's primary objective over the next 12 months is increasing the supply side of the marketplace by acquiring:

* Landlords
* Agents
* Property Managers
* Listings

The homepage will remain focused on renters searching for properties.

Dedicated acquisition landing pages will be created to convert landlords, agents, referrers, and student ambassadors.

---

# Goals

## Business Goals

* Increase landlord acquisition.
* Increase agent acquisition.
* Increase listing inventory.
* Launch referral acquisition program.
* Launch student ambassador program.

## User Goals

### Landlords

* Understand the value of listing on Ndotoni.
* Contact Ndotoni.
* Submit a property.

### Agents

* Understand partnership benefits.
* Register as a partner.

### Referrers

* Understand referral rewards.
* Submit landlord referrals.

### Students

* Learn about ambassador opportunities.
* Apply to join the program.

# Pages

## Landlord Page

### Route

`/landlords`

### Purpose

Convert landlords into property listers.

### Success Metric

* Landlord inquiries submitted.
* Landlord registrations.
* Listings created.

### Scope

#### Hero Section

Must include:

* Headline
* Subheadline
* Primary CTA
* Secondary CTA

#### Benefits Section

Must communicate:

* More tenant exposure.
* Reduced vacancy time.
* Time savings.
* Trust and verification.

#### How It Works Section

Must explain:

1. Submit property.
2. Ndotoni publishes listing.
3. Tenants inquire.
4. Ndotoni assists.
5. Landlord selects tenant.

#### Social Proof Section

Should display:

* Number of listings.
* Number of renters.
* Number of successful rentals.

#### CTA Section

Final conversion section encouraging property submission.

### Out of Scope

* Landlord dashboard.
* Property management tools.
* Messaging system.
* Payments.
* Authentication.

## Referral Program Page

### Route

`/refer`

### Purpose

Acquire landlords through referrals.

### Success Metric

* Referral submissions.
* Qualified referrals.
* Landlords onboarded via referrals.

### Scope

#### Hero Section

Must explain:

* Referral rewards.
* Program overview.

#### Reward Breakdown

Must explain:

* 2,000 reward for first listing.
* 10% reward on successful rental.

#### Process Section

Must explain:

1. Submit referral.
2. Verification.
3. Listing published.
4. Reward paid.
5. Rental completed.
6. Bonus reward paid.

#### Fraud Prevention Section

Must explain:

* New landlords only.
* Unique phone numbers.
* Verification requirements.
* Referral limits.

#### Referral Form

Required Fields:

* Referrer name
* Referrer phone number
* Landlord name
* Landlord phone number
* Property location
* Additional notes

### Out of Scope

* Automated payout systems.
* Referral dashboards.
* Referral analytics.
* Wallet functionality.

# Shared Requirements

## Design Requirements

All pages must:

* Follow existing Ndotoni branding.
* Be mobile-first.
* Support Swahili and English content.
* Include clear call-to-action buttons.
* Load quickly on mobile networks.

## Analytics Requirements

Track:

* Page views
* CTA clicks
* Form submissions
* Conversion rates

## SEO Requirements

Each page must have:

* Unique title
* Unique meta description
* Open Graph metadata
* Search-friendly URL structure

## Accessibility Requirements

* Mobile responsive
* Accessible buttons
* Readable typography
* Proper heading hierarchy
